KEY FEATURES
================================================================================

✓ 20+ Notification Types
  - Tasks (due, overdue, completed)
  - Documents (shared, version, verified, rejected)
  - Progress (milestone, phase, PACE™)
  - Team (member joined, mention)
  - Business Critical (cap table, board report, regulatory, SEDI)
  - System (alerts, warnings, subscription, expert response)

✓ 4 Notification Channels
  - Email (via Resend)
  - SMS (via Twilio)
  - Push (browser/mobile)
  - WhatsApp (via Twilio)

✓ 4 Delivery Frequencies
  - Real-time: Send immediately (unless in quiet hours)
  - Daily Digest: Queue for morning email
  - Weekly: Queue for weekly summary
  - Never: Disable notification type

✓ Timezone Support
  - 20+ IANA timezone strings
  - Automatic DST handling
  - Per-notification timezone override

✓ Quiet Hours (Do Not Disturb)
  - Global DND window (e.g., 22:00 - 08:00)
  - Per-notification-type override
  - Real-time notifications blocked
  - Digest notifications queued for next delivery
  - Critical notifications bypass DND

✓ Digest Scheduling
  - Configurable digest time (e.g., 09:00)
  - Timezone-aware delivery
  - Smart queuing during quiet hours
  - Multiple notification types per digest

✓ Per-Channel Control
  - Enable/disable each channel independently
  - Different defaults per notification type
  - SMS/WhatsApp only for critical types

✓ Smart Defaults
  - All channels enabled by default (except SMS/WhatsApp)
  - Real-time for urgent, digest for routine
  - Sensible per-type configuration

DATABASE SCHEMA
================================================================================

TABLE: notification_preferences
├─ id (UUID)
├─ user_id (TEXT) - FK to auth.users
├─ notification_type (VARCHAR) - Task/document/progress type
├─ email_enabled (BOOLEAN)
├─ sms_enabled (BOOLEAN)
├─ push_enabled (BOOLEAN)
├─ whatsapp_enabled (BOOLEAN)
├─ frequency (VARCHAR) - real_time/daily_digest/weekly/never
├─ quiet_hours_start (TIME, optional)
├─ quiet_hours_end (TIME, optional)
├─ quiet_hours_timezone (VARCHAR, optional)
└─ updated_at (TIMESTAMP)

TABLE: notification_settings
├─ id (UUID)
├─ user_id (TEXT, UNIQUE) - FK to auth.users
├─ digest_time (VARCHAR) - HH:mm format
├─ digest_timezone (VARCHAR) - IANA timezone
├─ do_not_disturb_start (VARCHAR) - HH:mm format
├─ do_not_disturb_end (VARCHAR) - HH:mm format
├─ do_not_disturb_timezone (VARCHAR) - IANA timezone
├─ created_at (TIMESTAMP)
└─ updated_at (TIMESTAMP)

INDEXES:
├─ notification_preferences.user_id
└─ notification_settings.user_id

API ENDPOINTS
================================================================================

GET /api/notifications/preferences
  ├─ Gets all preferences and global settings
  ├─ Response: { preferences: {...}, settings: {...} }
  └─ Auth: Required

GET /api/notifications/preferences/:type
  ├─ Gets preference for specific notification type
  ├─ Params: type (e.g., 'task_due')
  └─ Auth: Required

PUT /api/notifications/preferences/:type
  ├─ Updates preference for specific type (partial)
  ├─ Body: { emailEnabled: bool, frequency: string, ... }
  └─ Auth: Required

POST /api/notifications/preferences
  ├─ Bulk update preferences and settings
  ├─ Body: { preferences: [...], settings: {...} }
  └─ Auth: Required

POST /api/notifications/digest-time
  ├─ Set digest time and timezone
  ├─ Body: { digestTime: "HH:mm", digestTimezone: "..." }
  └─ Auth: Required
