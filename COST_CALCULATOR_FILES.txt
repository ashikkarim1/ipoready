7. DATABASE MIGRATION
   📄 migrations/020_cost_calculator.sql (2.8 KB)
   
   Creates 3 tables:
   
   a) cost_calculations (primary table)
      Columns: id, company_id, calculation_data, company_revenue, 
               selected_exchange, complexity_level, timeline_months, 
               total_cost, cost_breakdown, benchmarks, notes, 
               created_at, updated_at
      Indexes: company_id, created_at, exchange
      
   b) cost_calculation_scenarios (optional, future use)
      Columns: id, cost_calculation_id, scenario_name, scenario_data, 
               total_cost, description, is_selected, created_at, updated_at
      Indexes: calculation_id, is_selected
      
   c) cost_savings_strategies (optional, future use)
      Columns: id, cost_calculation_id, category, strategy_name, 
               description, potential_savings, implementation_difficulty, 
               estimated_timeline, implementation_status, created_at, updated_at
      Indexes: calculation_id, category
   
   Features:
   - Primary keys (UUIDs)
   - Foreign keys with cascade delete
   - JSONB columns for flexible storage
   - Indexes for fast queries
   - Automatic timestamp triggers
   - Normalized schema design
