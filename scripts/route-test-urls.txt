# ============================================================================
# TESTING CHECKLIST
# ============================================================================

For each route, verify:

1. [ ] Page loads without error (check HTTP status)
2. [ ] No console errors (F12 -> Console tab)
3. [ ] No console warnings that indicate problems
4. [ ] Page styling is consistent (light theme)
5. [ ] Navigation is functional
6. [ ] Forms (if present) are interactive
7. [ ] Data loads properly (if applicable)
8. [ ] Images load correctly
9. [ ] No layout shifts (CLS)
10. [ ] Mobile responsive (test with device toolbar if needed)

When testing protected routes:
- [ ] Must be logged in
- [ ] User context available
- [ ] Database queries complete
- [ ] No authentication errors

# ============================================================================
# BROWSER CONSOLE CHECKS
# ============================================================================

Open DevTools (F12 or Cmd+Option+I) and check:

1. Console tab:
   - [ ] No red errors
   - [ ] No warnings about missing components
   - [ ] No hydration mismatch warnings
   - [ ] No infinite loops in console

2. Network tab:
   - [ ] All requests return 200 status (or expected codes)
   - [ ] No 404 errors for resources
   - [ ] API calls complete successfully
   - [ ] Response times reasonable (< 500ms for most)

3. Application tab (for cookies/storage):
   - [ ] NextAuth session cookie present (if authenticated)
   - [ ] SessionToken stored correctly
   - [ ] No sensitive data exposed
   - [ ] localStorage/sessionStorage clean

# ============================================================================
# COMMON ISSUES TO WATCH FOR
# ============================================================================

1. Hydration Mismatches:
   - Error: "Text content did not match"
   - Solution: Check for client/server component mismatch

2. Missing Environment Variables:
   - Error: "DATABASE_URL is not defined"
   - Solution: Check .env.local is configured correctly

3. Port Already in Use:
   - Error: "Port 3000 is already in use"
   - Solution: Kill existing process or use different port

4. Database Connection Errors:
   - Error: "Failed to connect to database"
   - Solution: Verify Neon connection string in .env.local

5. Build Failures:
   - Error: "TypeScript compilation failed"
   - Solution: Run npm run build locally to see full error

# ============================================================================
# PERFORMANCE TESTING (Optional)
# ============================================================================

To test performance with Lighthouse:

1. Open DevTools (F12)
2. Click "Lighthouse" tab
3. Select "Desktop" or "Mobile"
4. Click "Analyze page load"
5. Wait for report
6. Target scores:
   - Performance: > 75
   - Accessibility: > 90
   - Best Practices: > 90
   - SEO: > 90
