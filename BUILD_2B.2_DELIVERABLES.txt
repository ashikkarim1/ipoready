API ENDPOINTS (137 lines)
════════════════════════════════════════════════════════════════════════════════

Route Handler:
FILE: src/app/api/compliance/consents/route.ts (137 lines)

GET Endpoint:
  - Fetch consents for a company
  - Optional filtering by entity_type or status
  - Returns array of consent records
  - Authentication required (NextAuth)

POST Endpoint:
  - Create new consent record
  - Update existing consent (if id provided)
  - Validates required fields
  - Returns created/updated record
  - Authentication required

DELETE Endpoint:
  - Remove consent record
  - Requires consent id and company_id
  - Returns { success: true }
  - Authentication required
